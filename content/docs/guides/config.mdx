このガイドでは、Starweft の TOML 設定ファイルの構造と主要な設定項目を説明します。

## 設定ファイルの場所

設定ファイルはデータディレクトリ内の `config.toml` です。

| プラットフォーム | デフォルトパス |
|----------------|---------------|
| macOS / Linux | `~/.starweft/config.toml` |
| Windows | `%LOCALAPPDATA%\starweft\config.toml` |

`--data-dir` オプションでデータディレクトリを指定した場合は、そのディレクトリ内の `config.toml` が使用されます。

## 設定の確認と検証

### `config show`

現在の設定を表示します。シークレット値は自動的にマスクされます。

```bash
starweft config show --data-dir ~/.starweft
```

JSON 形式で出力:

```bash
starweft config show --data-dir ~/.starweft --json
```

### `config validate`

設定ファイルの妥当性を検証し、エラーと警告を報告します。

```bash
starweft config validate --data-dir ~/.starweft
```

検証される項目:
- リッスンアドレスの形式
- データディレクトリの存在
- Identity 鍵ファイルの存在とパーミッション
- プロトコルバージョンとスキーマバージョンの互換性
- OpenClaw バイナリの存在
- `capability_version` と `bridge_capability_version` の一致

## 設定セクション一覧

```
config.toml
├── [node]           — ノードの基本設定
├── [identity]       — 鍵ファイルのパス
├── [discovery]      — ピア検出・mDNS・レジストリ
├── [p2p]            — トランスポート設定
├── [ledger]         — SQLite データベースのパス
├── [openclaw]       — OpenClaw ブリッジ設定
├── [compatibility]  — プロトコルバージョン
├── [owner]          — Owner ロール固有の設定
├── [worker]         — Worker ロール固有の設定
├── [observation]    — プランニング・評価エンジン
├── [logs]           — ログローテーション
└── [artifacts]      — 成果物の保存設定
```

## `[node]` セクション

ノードの基本設定です。

```toml
[node]
role = "owner"
display_name = "owner-node"
data_dir = "~/.starweft"
listen = ["/unix//home/user/.starweft/mailbox.sock"]
log_level = "info"
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `role` | string | ノードの役割: principal / owner / worker / relay |
| `display_name` | string | ノードの表示名 |
| `data_dir` | string | データディレクトリのパス |
| `listen` | string[] | リッスンアドレスのリスト |
| `log_level` | string | ログレベル: trace / debug / info / warn / error |

### リッスンアドレスの形式

| トランスポート | 形式 | 例 |
|---------------|------|-----|
| Unix ソケット | `/unix/<path>` | `/unix//home/user/.starweft/mailbox.sock` |
| TCP | `/ip4/<addr>/tcp/<port>` | `/ip4/0.0.0.0/tcp/9100` |
| IPv6 TCP | `/ip6/<addr>/tcp/<port>` | `/ip6/::1/tcp/9100` |

## `[identity]` セクション

Ed25519 鍵ファイルのパスを指定します。

```toml
[identity]
actor_key_path = "~/.starweft/identity/actor_key"
stop_authority_key_path = "~/.starweft/identity/stop_authority_key"
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `actor_key_path` | string? | メッセージ署名に使用する Ed25519 秘密鍵のパス |
| `stop_authority_key_path` | string? | 停止命令の署名に使用する鍵のパス（principal のみ） |

<Callout type="info">
  `stop_authority_key_path` は principal ロールでのみ自動生成されます。他のロールでは `null` です。
</Callout>

## `[discovery]` セクション

ピア検出とレジストリの設定です。

```toml
[discovery]
seeds = ["/ip4/192.168.1.20/tcp/9100/p2p/12D3Koo..."]
auto_discovery = true
mdns = false
registry_url = ""
registry_ttl_sec = 300
registry_heartbeat_sec = 60
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `seeds` | string[] | 起動時に接続するシードピアのマルチアドレス |
| `auto_discovery` | bool | CapabilityQuery/Advertisement の自動交換 (default: true) |
| `mdns` | bool | mDNS によるローカルネットワーク自動検出 (default: false) |
| `registry_url` | string? | ピアレジストリサーバーの URL |
| `registry_ttl_sec` | u64 | レジストリ登録の TTL（秒） (default: 300) |
| `registry_heartbeat_sec` | u64 | レジストリへのハートビート間隔（秒） (default: 60) |
| `registry_shared_secret` | string? | レジストリ認証用の共有シークレット |
| `registry_shared_secret_env` | string? | 共有シークレットを読み取る環境変数名 |

<Callout type="warning">
  `registry_shared_secret` を設定ファイルに直接記載する場合、ファイルのパーミッションが 0600（所有者のみ読み書き）であることを確認してください。`config show` で表示する際は自動的にマスクされます。
</Callout>

## `[p2p]` セクション

トランスポートの種類と接続設定です。

```toml
[p2p]
transport = "local_mailbox"
relay_enabled = true
direct_preferred = true
max_peers = 128
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `transport` | string | local_mailbox（Unix デフォルト）または libp2p（Windows デフォルト） |
| `relay_enabled` | bool | relay ノード経由の通信を許可 (default: true) |
| `direct_preferred` | bool | 直接接続を優先 (default: true) |
| `max_peers` | u16 | 最大ピア数 (default: 128) |

## `[openclaw]` セクション

OpenClaw ブリッジの設定です。Worker ノードで使用します。

```toml
[openclaw]
enabled = false
bin = "openclaw"
working_dir = "/home/user/workspace"
timeout_sec = 3600
capability_version = "openclaw.execution.v1"
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `enabled` | bool | OpenClaw ブリッジを有効にする (default: false) |
| `bin` | string | OpenClaw バイナリのパス (default: openclaw) |
| `working_dir` | string? | サブプロセスの作業ディレクトリ |
| `timeout_sec` | u64 | 実行タイムアウト（秒） (default: 3600) |
| `capability_version` | string | capability 識別子 (default: openclaw.execution.v1) |

## `[compatibility]` セクション

プロトコルバージョンとスキーマバージョンの設定です。

```toml
[compatibility]
protocol_version = "starweft/0.1"
schema_version = "starweft-store/2"
bridge_capability_version = "openclaw.execution.v1"
allow_legacy_protocols = false
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `protocol_version` | string | メッセージプロトコルバージョン |
| `schema_version` | string | SQLite スキーマバージョン |
| `bridge_capability_version` | string | OpenClaw ブリッジの capability バージョン |
| `allow_legacy_protocols` | bool | 古いプロトコルバージョンを許可 (default: false) |

<Callout type="warning">
  `openclaw.capability_version` と `bridge_capability_version` が一致しない場合、ノード起動時にエラーが発生します。
</Callout>

## `[owner]` セクション

Owner ロール固有の設定です。タスク失敗時のリトライ動作を制御します。

```toml
[owner]
max_retry_attempts = 8
retry_cooldown_ms = 250
retry_strategy = "rule_based"

[[owner.retry_rules]]
pattern = "timeout"
action = "retry_same_worker"
reason = "transient timeout"

[[owner.retry_rules]]
pattern = "process failed"
action = "retry_different_worker"
reason = "transient execution failure"

[[owner.retry_rules]]
pattern = "capability mismatch"
action = "no_retry"
reason = "permanent task/input failure"
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `max_retry_attempts` | u64 | 最大リトライ回数 (default: 8) |
| `retry_cooldown_ms` | u64 | リトライ間隔（ミリ秒） (default: 250) |
| `retry_strategy` | string | リトライ戦略: rule_based (default: rule_based) |
| `retry_rules` | array | エラーパターンとリトライアクションのルール配列 |

### リトライアクション

| アクション | 説明 |
|-----------|------|
| `retry_same_worker` | 同じ Worker で再試行 |
| `retry_different_worker` | 別の Worker で再試行 |
| `no_retry` | リトライしない |

## `[worker]` セクション

Worker ロール固有の設定です。

```toml
[worker]
accept_join_offers = true
max_active_tasks = 1
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `accept_join_offers` | bool | JoinOffer を自動で受け入れる (default: true) |
| `max_active_tasks` | u64 | 同時実行可能なタスク数 (default: 1) |

## `[observation]` セクション

タスクプランニングと評価エンジンの設定です。

```toml
[observation]
cache_snapshots = true
cache_ttl_sec = 30
max_planned_tasks = 6
min_task_objective_chars = 48
planner = "heuristic"
evaluator = "heuristic"
planner_bin = ""
planner_working_dir = ""
planner_timeout_sec = 120
planner_capability_version = "openclaw.plan.v1"
planner_fallback_to_heuristic = true
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `cache_snapshots` | bool | スナップショットのキャッシュ (default: true) |
| `cache_ttl_sec` | u64 | キャッシュの有効期間（秒） (default: 30) |
| `max_planned_tasks` | usize | 1 ビジョンから生成する最大タスク数 (default: 6) |
| `min_task_objective_chars` | usize | タスク目的の最小文字数 (default: 48) |
| `planner` | string | heuristic / openclaw / openclaw_worker (default: heuristic) |
| `evaluator` | string | heuristic (default: heuristic) |
| `planner_bin` | string? | openclaw プランナーのバイナリパス |
| `planner_working_dir` | string? | プランナーの作業ディレクトリ |
| `planner_timeout_sec` | u64 | プランナーのタイムアウト（秒） (default: 120) |
| `planner_capability_version` | string | プランナーの capability バージョン (default: openclaw.plan.v1) |
| `planner_fallback_to_heuristic` | bool | OpenClaw プランナー失敗時にヒューリスティックへフォールバック (default: true) |

## `[logs]` セクション

ログローテーションの設定です。

```toml
[logs]
rotate_max_bytes = 1048576
max_archives = 5
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `rotate_max_bytes` | u64 | ログファイルの最大サイズ（バイト） (default: 1048576 = 1MB) |
| `max_archives` | usize | 保持するアーカイブファイル数 (default: 5) |

## `[artifacts]` セクション

タスク成果物の保存設定です。

```toml
[artifacts]
dir = "~/.starweft/artifacts"
max_files = 256
max_age_sec = 604800
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `dir` | string | 成果物の保存ディレクトリ |
| `max_files` | usize | 最大保存ファイル数 (default: 256) |
| `max_age_sec` | u64 | 成果物の最大保持期間（秒） (default: 604800 = 7日) |

## ロール別の推奨設定

### Principal

```toml
[node]
role = "principal"
log_level = "info"

[identity]
actor_key_path = "~/.starweft/identity/actor_key"
stop_authority_key_path = "~/.starweft/identity/stop_authority_key"
```

Principal はビジョンの投入と停止命令の発行のみを行います。`stop_authority_key_path` は principal ロールでのみ必要です。

### Owner

```toml
[node]
role = "owner"

[owner]
max_retry_attempts = 8
retry_cooldown_ms = 250

[observation]
planner = "heuristic"
max_planned_tasks = 6
cache_snapshots = true
```

Owner はタスクの分解・配布・評価を担当します。`max_planned_tasks` はビジョンの複雑さに応じて調整してください。

### Worker

```toml
[node]
role = "worker"

[openclaw]
enabled = true
bin = "/usr/local/bin/openclaw"
timeout_sec = 3600

[worker]
accept_join_offers = true
max_active_tasks = 1
```

Worker はタスクの実行を担当します。`max_active_tasks` はマシンリソースに応じて調整してください。

### Relay

```toml
[node]
role = "relay"
listen = ["/ip4/0.0.0.0/tcp/9300"]

[p2p]
transport = "libp2p"
```

Relay はメッセージ転送のみを行います。identity や OpenClaw の設定は不要です。

## よくある設定シナリオ

### シングルマシンの開発環境

```toml
[p2p]
transport = "local_mailbox"

[discovery]
mdns = false
```

同一マシン上で複数ノードを起動する場合は、各ノードに異なる `--data-dir` を指定し、Unix ソケットで通信します。

### LAN 内のマルチマシン環境

```toml
[p2p]
transport = "libp2p"

[discovery]
mdns = true
auto_discovery = true
```

同一 LAN 内のノードは mDNS で自動検出されます。

### インターネット越しの分散環境

```toml
[p2p]
transport = "libp2p"
relay_enabled = true

[discovery]
mdns = false
seeds = ["/ip4/203.0.113.10/tcp/9300/p2p/12D3Koo..."]
```

NAT の背後にあるノードは relay を経由して通信します。mDNS は無効にしてください。

## セキュリティに関する注意

- 設定ファイルは **0600 パーミッション**（所有者のみ読み書き）で保存されます
- `registry_shared_secret` は環境変数経由（`registry_shared_secret_env`）での指定を推奨します
- `config show` はシークレット値を自動的に `<redacted>` にマスクして表示します
- Windows 環境では、ファイルに隠し属性が設定され、ACL で現在のユーザーのみにアクセスが制限されます
