## 概要

Starweft の設定は `config.toml` で管理されます。`starweft init` で自動生成され、デフォルトではデータディレクトリ直下 (`~/.starweft/config.toml`) に配置されます。

```bash
# 現在の設定を表示
starweft config show

# 設定の検証
starweft config validate
```

<Callout type="info">
`config show` は秘密情報 (registry_shared_secret など) をマスクして表示します。
</Callout>

---

## [node]

ノードの基本設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `role` | string | - | ノードの役割。principal / owner / worker / relay のいずれか |
| `display_name` | string | `"<role>-node"` | ノードの表示名 |
| `data_dir` | string | `"~/.starweft"` | データディレクトリのパス。~ はホームディレクトリに展開される |
| `listen` | string[] | 自動生成 | リッスンアドレスのリスト (multiaddr 形式) |
| `log_level` | string | `"info"` | ログレベル。trace / debug / info / warn / error |

```toml
[node]
role = "worker"
display_name = "worker-01"
data_dir = "~/.starweft-worker"
listen = ["/unix/~/.starweft-worker/mailbox.sock"]
log_level = "info"
```

<Callout type="tip">
Unix ではデフォルトで Unix ソケット (`/unix/...`) が使われます。Windows では TCP (`/ip4/127.0.0.1/tcp/0`) が使われます。
</Callout>

## [identity]

Ed25519 鍵ファイルのパスを指定します。`starweft identity create` で自動設定されます。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `actor_key_path` | string \| null | null | actor 鍵ファイルのパス。署名に使用 |
| `stop_authority_key_path` | string \| null | null | stop authority 鍵ファイルのパス。principal が stop 命令に署名する際に使用 |

```toml
[identity]
actor_key_path = "~/.starweft/identity/actor_key"
stop_authority_key_path = "~/.starweft/identity/stop_authority_key"
```

<Callout type="warning">
鍵ファイルは 0600 パーミッション (Unix) で保護されます。手動で作成した場合はパーミッションを確認してください。
</Callout>

## [discovery]

ピアディスカバリの設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `seeds` | string[] | `[]` | 初期接続先ピアの multiaddr リスト |
| `auto_discovery` | bool | `true` | ピアからの自動ディスカバリを有効化 |
| `mdns` | bool | libp2p 時 true | mDNS によるローカルネットワークディスカバリ (v0.3.0 で追加。transport=libp2p 時のデフォルト) |
| `registry_url` | string \| null | null | レジストリサーバーの URL |
| `registry_ttl_sec` | u64 | `300` | レジストリエントリの TTL (秒) |
| `registry_heartbeat_sec` | u64 | `60` | レジストリへのハートビート間隔 (秒) |
| `registry_shared_secret` | string \| null | null | レジストリ認証用の共有シークレット (直接指定) |
| `registry_shared_secret_env` | string \| null | null | 共有シークレットを格納する環境変数名 |

```toml
[discovery]
seeds = ["/unix/~/.starweft-owner/mailbox.sock"]
auto_discovery = true
mdns = false
registry_url = "http://127.0.0.1:7777"
registry_ttl_sec = 300
registry_heartbeat_sec = 60
```

<Callout type="tip">
`registry_shared_secret` を `config.toml` に直接書く代わりに `registry_shared_secret_env` で環境変数名を指定すると、設定ファイルにシークレットを残さずに済みます。
</Callout>

## [p2p]

P2P トランスポートの設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `transport` | string | Unix: `"local_mailbox"` / Windows: `"libp2p"` | トランスポート種別。local_mailbox (Unix ソケット) または libp2p (TCP) |
| `relay_enabled` | bool | `true` | relay ノード経由の通信を有効化 |
| `direct_preferred` | bool | `true` | 直接接続を優先 |
| `max_peers` | u16 | `128` | 接続可能なピアの最大数 |

```toml
[p2p]
transport = "local_mailbox"
relay_enabled = true
direct_preferred = true
max_peers = 128
```

## [ledger]

イベントソーシング用 SQLite データベースの設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `path` | string | `"<data_dir>/ledger/node.db"` | SQLite データベースファイルのパス |

```toml
[ledger]
path = "~/.starweft/ledger/node.db"
```

## [openclaw]

OpenClaw サブプロセスの設定です。主に **worker** ロールで使用します。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `enabled` | bool | `false` | OpenClaw 統合を有効化 |
| `bin` | string | `"openclaw"` | OpenClaw バイナリのパスまたはコマンド名 |
| `working_dir` | string \| null | null | OpenClaw サブプロセスの作業ディレクトリ |
| `timeout_sec` | u64 | `3600` | タスク実行タイムアウト (秒) |
| `capability_version` | string | `"openclaw.execution.v1"` | OpenClaw ブリッジの capability バージョン |

```toml
[openclaw]
enabled = true
bin = "/usr/local/bin/openclaw"
working_dir = "/home/user/workspace"
timeout_sec = 3600
capability_version = "openclaw.execution.v1"
```

## [compatibility]

プロトコルとスキーマのバージョン互換性を制御します。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `protocol_version` | string | `"starweft/0.1"` | メッセージプロトコルのバージョン |
| `schema_version` | string | 自動 (現在の store バージョン) | ストアスキーマのバージョン |
| `bridge_capability_version` | string | `"openclaw.execution.v1"` | OpenClaw ブリッジの capability バージョン。openclaw.capability_version と一致する必要がある |
| `allow_legacy_protocols` | bool | `false` | 旧バージョンプロトコルのメッセージを許容 |

```toml
[compatibility]
protocol_version = "starweft/0.1"
schema_version = "starweft-store/3"
bridge_capability_version = "openclaw.execution.v1"
allow_legacy_protocols = false
```

<Callout type="warning">
`bridge_capability_version` と `openclaw.capability_version` が一致しない場合、ランタイム起動時に `[E_BRIDGE_CAPABILITY_VERSION]` エラーが発生します。
</Callout>

## [owner]

**owner** ロール固有の設定です。タスク失敗時のリトライポリシーを制御します。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `max_retry_attempts` | u64 | `8` | 最大リトライ回数 |
| `retry_cooldown_ms` | u64 | `250` | リトライ間隔 (ミリ秒) |
| `retry_strategy` | string | `"rule_based"` | リトライ戦略。現在は rule_based のみ |
| `retry_rules` | OwnerRetryRule[] | デフォルトルールセット | パターンベースのリトライルール (下記参照) |

### retry_rules

各ルールは `pattern` (エラーメッセージの部分一致)、`action` (リトライ動作)、`reason` (ログ用理由) で構成されます。

**action の値:**
- `retry_same_worker` -- 同じ worker でリトライ
- `retry_different_worker` -- 別の worker でリトライ
- `no_retry` -- リトライしない

```toml
[owner]
max_retry_attempts = 8
retry_cooldown_ms = 250
retry_strategy = "rule_based"

[[owner.retry_rules]]
pattern = "timeout"
action = "retry_same_worker"
reason = "transient timeout"

[[owner.retry_rules]]
pattern = "process failed"
action = "retry_different_worker"
reason = "transient execution failure"

[[owner.retry_rules]]
pattern = "capability mismatch"
action = "no_retry"
reason = "permanent task/input failure"
```

<Accordion title="デフォルトリトライルール一覧">

| pattern | action | reason |
|---|---|---|
| `timeout` | `retry_same_worker` | transient timeout |
| `timed out` | `retry_same_worker` | transient timeout |
| `process failed` | `retry_different_worker` | transient execution failure |
| `worker overloaded` | `retry_different_worker` | transient execution failure |
| `worker unavailable` | `retry_different_worker` | transient execution failure |
| `capability mismatch` | `no_retry` | permanent task/input failure |
| `invalid input` | `no_retry` | permanent task/input failure |
| `schema` | `no_retry` | permanent task/input failure |
| `unauthorized` | `no_retry` | permanent task/input failure |

</Accordion>

## [worker]

**worker** ロール固有の設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `accept_join_offers` | bool | `true` | owner からの JoinOffer を受け入れるか |
| `max_active_tasks` | u64 | `1` | 同時実行可能なタスクの最大数 |

```toml
[worker]
accept_join_offers = true
max_active_tasks = 1
```

## [observation]

プランニングと評価の設定です。owner ロールがビジョンをタスクに分解する際に使用します。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `planner` | string | `"heuristic"` | プランニング戦略。heuristic / openclaw / openclaw_worker |
| `evaluator` | string | `"heuristic"` | 評価戦略。現在は heuristic のみ |
| `cache_snapshots` | bool | `true` | スナップショットをキャッシュ |
| `cache_ttl_sec` | u64 | `30` | キャッシュの TTL (秒) |
| `max_planned_tasks` | usize | `6` | 1 ビジョンから生成するタスクの最大数 |
| `min_task_objective_chars` | usize | `48` | タスク目標テキストの最小文字数 |
| `planner_bin` | string \| null | null | プランナー用 OpenClaw バイナリ (openclaw/openclaw_worker 戦略時) |
| `planner_working_dir` | string \| null | null | プランナーの作業ディレクトリ |
| `planner_timeout_sec` | u64 | `120` | プランナーのタイムアウト (秒) |
| `planner_capability_version` | string | `"openclaw.plan.v1"` | プランナーの capability バージョン |
| `planner_fallback_to_heuristic` | bool | `true` | OpenClaw プランナー失敗時にヒューリスティックへフォールバック |

```toml
[observation]
planner = "heuristic"
evaluator = "heuristic"
cache_snapshots = true
cache_ttl_sec = 30
max_planned_tasks = 6
min_task_objective_chars = 48
planner_timeout_sec = 120
planner_capability_version = "openclaw.plan.v1"
planner_fallback_to_heuristic = true
```

<Accordion title="planner 戦略の比較">

| 戦略 | 実行場所 | 説明 |
|---|---|---|
| `heuristic` | ローカル (owner) | ビジョンテキストからルールベースでタスクを分解。LLM 不要 |
| `openclaw` | ローカル (owner) | ローカルの OpenClaw サブプロセスでプランニング |
| `openclaw_worker` | リモート (worker) | worker ノードにプランニングタスクを委任 |

</Accordion>

## [logs]

ランタイムログのローテーション設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `rotate_max_bytes` | u64 | `1048576` | ログファイルの最大サイズ (bytes)。デフォルトは 1 MB |
| `max_archives` | usize | `5` | 保持するアーカイブファイルの最大数 |

```toml
[logs]
rotate_max_bytes = 1048576
max_archives = 5
```

## [artifacts]

タスク実行で生成されるアーティファクトの保存設定です。

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `dir` | string | `"<data_dir>/artifacts"` | アーティファクト保存ディレクトリ |
| `max_files` | usize | `256` | 保持するアーティファクトファイルの最大数 |
| `max_age_sec` | u64 | `604800` | アーティファクトの最大保持期間 (秒)。デフォルトは 7 日 |

```toml
[artifacts]
dir = "~/.starweft/artifacts"
max_files = 256
max_age_sec = 604800
```

## 設定ファイルの完全な例

```toml
[node]
role = "worker"
display_name = "worker-01"
data_dir = "~/.starweft-worker"
listen = ["/unix/~/.starweft-worker/mailbox.sock"]
log_level = "info"

[identity]
actor_key_path = "~/.starweft-worker/identity/actor_key"

[discovery]
seeds = ["/unix/~/.starweft-owner/mailbox.sock"]
auto_discovery = true
mdns = false

[p2p]
transport = "local_mailbox"
relay_enabled = true
direct_preferred = true
max_peers = 128

[ledger]
path = "~/.starweft-worker/ledger/node.db"

[openclaw]
enabled = true
bin = "openclaw"
timeout_sec = 3600
capability_version = "openclaw.execution.v1"

[compatibility]
protocol_version = "starweft/0.1"
bridge_capability_version = "openclaw.execution.v1"
allow_legacy_protocols = false

[owner]
max_retry_attempts = 8
retry_cooldown_ms = 250
retry_strategy = "rule_based"

[worker]
accept_join_offers = true
max_active_tasks = 1

[observation]
planner = "heuristic"
evaluator = "heuristic"
cache_snapshots = true
cache_ttl_sec = 30

[logs]
rotate_max_bytes = 1048576
max_archives = 5

[artifacts]
dir = "~/.starweft-worker/artifacts"
max_files = 256
max_age_sec = 604800
```
