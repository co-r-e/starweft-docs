## init

ノードのデータディレクトリと `config.toml` を初期化します。

```bash
starweft init --role <ROLE> [OPTIONS]
```

<ApiTable data={[
  { flag: "--role", type: "principal | owner | worker | relay", default: "-", description: "必須。ノードの役割" },
  { flag: "--data-dir", type: "path", default: "~/.starweft", description: "データディレクトリのパス" },

## vision

ビジョン (目標) の投入とプランニングプレビューを行います。

### vision submit

ビジョンを owner に送信し、タスク分解を開始します。**principal ロール専用**です。

```bash
starweft vision submit --title <TITLE> [OPTIONS]
```

<ApiTable data={[
  { flag: "--data-dir", type: "path", default: "~/.starweft", description: "データディレクトリのパス" },
  { flag: "--title", type: "string", default: "-", description: "必須。ビジョンのタイトル" },
  { flag: "--text", type: "string", default: "-", description: "ビジョンのテキスト本文。--file と排他" },
  { flag: "--file", type: "path", default: "-", description: "ビジョンテキストを読み込むファイル。--text と排他" },
  { flag: "--constraint", type: "key=value[]", default: "-", description: "制約条件 (複数指定可: budget_mode, human_intervention, allow_external_agents)" },
  { flag: "--owner", type: "string", default: "自動解決", description: "送信先 owner の actor ID" },
  { flag: "--dry-run", type: "bool", default: "false", description: "プランニングプレビューのみ実行" },
  { flag: "--approve", type: "TOKEN", default: "-", description: "プレビューで取得した approval token を指定して submit" },
  { flag: "--missing-only", type: "bool", default: "false", description: "--dry-run と併用。不足情報のみ表示" },
  { flag: "--json", type: "bool", default: "false", description: "JSON 形式で出力" },
]} />

<Callout type="info">
`human_intervention=required` の制約を持つビジョンは、先に `--dry-run` でプレビューを確認し、表示される `--approve` トークンを使って submit する必要があります。
</Callout>

<Terminal title="例: ビジョンを dry-run で確認してから submit">
```bash
# Step 1: プレビュー
starweft vision submit --title "新機能開発" --text "..." --dry-run

# Step 2: approve トークンで submit
starweft vision submit --title "新機能開発" --text "..." --approve <TOKEN>
```
</Terminal>

## stop

```bash
starweft stop --reason-code <CODE> --reason <TEXT> --yes [OPTIONS]

  { flag: "--data-dir", type: "path", default: "~/.starweft", description: "データディレクトリのパス" },

  { flag: "--yes", type: "bool", default: "false", description: "必須。停止を確認" },
  { flag: "--json", type: "bool", default: "false", description: "JSON 形式で出力" },

```

## wait

指定した条件が満たされるまでポーリングで待機します。

```bash
starweft wait [OPTIONS]
```

<ApiTable data={[
  { flag: "--data-dir", type: "path", default: "~/.starweft", description: "データディレクトリのパス" },
  { flag: "--vision", type: "string", default: "-", description: "対象 vision ID" },
  { flag: "--project", type: "string", default: "-", description: "対象プロジェクト ID" },
  { flag: "--task", type: "string", default: "-", description: "対象タスク ID" },
  { flag: "--stop", type: "string", default: "-", description: "対象 stop ID" },
  { flag: "--until", type: "string", default: "-", description: "待機条件 (下記参照)" },
  { flag: "--timeout-sec", type: "u64", default: "120", description: "タイムアウト (秒)" },
  { flag: "--interval-ms", type: "u64", default: "500", description: "ポーリング間隔 (ミリ秒)" },
  { flag: "--json", type: "bool", default: "false", description: "JSON 形式で出力" },
]} />

`--until` に指定できる値はスコープによって異なります。

<Accordion title="--until の有効な値">

- **`--vision`**: `project_created`, `active`, `stopped`
- **`--project`**: `available`, `active`, `approval_applied`, `stopping`, `stopped`
- **`--stop`**: `ordered`, `stopping`, `stopped`

</Accordion>

<Callout type="info">
`--vision`, `--project`, `--task`, `--stop` はいずれか 1 つだけ指定してください。
</Callout>
