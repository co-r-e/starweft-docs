## グローバルフラグ

すべてのサブコマンドで利用可能なグローバルフラグです。

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `-v, --verbose` | count | `0` | ログ出力を増やす (-v: debug, -vv: trace) |
| `-q, --quiet` | count | `0` | ログ出力を減らす (-q: warn, -qq: error)。--verbose と排他 |

## init

ノードのデータディレクトリと `config.toml` を初期化します。

```bash
starweft init --role <ROLE> [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--role` | principal \| owner \| worker \| relay | - | 必須。ノードの役割 |
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--display-name` | string | `<role>-node` | ノードの表示名 |
| `--listen` | multiaddr[] | 自動生成 | リッスンアドレス (複数指定可) |
| `--no-identity` | bool | `false` | identity create の案内を抑制 |
| `--force` | bool | `false` | 既存 config.toml を上書き |

<Terminal title="例: worker ノードを初期化">
```bash
starweft init --role worker --data-dir ~/.starweft-worker
```
</Terminal>

## run

ノードをフォアグラウンドで起動し、inbox ポーリングとメッセージ処理を開始します。

```bash
starweft run [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--role` | principal \| owner \| worker \| relay | config の値 | 起動時に role を上書き |
| `--foreground` | bool | `false` | フォアグラウンド実行を明示 |
| `--log-level` | string | config の値 | ログレベルを上書き (debug, info, warn, error, trace) |

<Terminal title="例: owner ノードを起動">
```bash
starweft run --data-dir ~/.starweft-owner
```
</Terminal>

## identity

Ed25519 鍵ペアの生成と表示を行います。

### identity create

actor_key (および principal の場合は stop_authority_key) を生成し、ローカル identity をストアに登録します。

```bash
starweft identity create [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--principal` | bool | `false` | principal 以外のロールでも stop_authority_key を生成 |
| `--force` | bool | `false` | 既存の鍵を上書き |

### identity show

ローカル identity 情報を表示します。

```bash
starweft identity show [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--json` | bool | `false` | JSON 形式で出力 |

<Terminal title="例: identity を確認">
```bash
starweft identity show --json
```
</Terminal>

## peer

ピアノードのアドレスと公開鍵を管理します。

### peer add

ピアのマルチアドレスを登録し、discovery seeds に追加します。

```bash
starweft peer add <MULTIADDR> [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `<MULTIADDR>` | string | - | 必須。ピアのマルチアドレス |
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--label` | string | - | ピアのラベル (表示用) |
| `--actor-id` | string | 自動生成 | ピアの actor ID |
| `--node-id` | string | 自動生成 | ピアの node ID |
| `--public-key` | string | - | ピアの公開鍵 (Base64) |
| `--public-key-file` | path | - | ピアの公開鍵ファイル。--public-key と排他 |
| `--stop-public-key` | string | - | ピアの stop authority 公開鍵 (Base64) |
| `--stop-public-key-file` | path | - | ピアの stop authority 公開鍵ファイル |
| `--capability` | string[] | - | ピアの capability (複数指定可) |

<Terminal title="例: owner ピアを追加">
```bash
starweft peer add /unix/~/.starweft-owner/mailbox.sock \
  --label owner \
  --public-key "BASE64_PUBLIC_KEY"
```
</Terminal>

| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |

## vision

ビジョン (目標) の投入とプランニングプレビューを行います。

### vision submit

ビジョンを owner に送信し、タスク分解を開始します。**principal ロール専用**です。

```bash
starweft vision submit --title <TITLE> [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--title` | string | - | 必須。ビジョンのタイトル |
| `--text` | string | - | ビジョンのテキスト本文。--file と排他 |
| `--file` | path | - | ビジョンテキストを読み込むファイル。--text と排他 |
| `--constraint` | key=value[] | - | 制約条件 (複数指定可: budget_mode, human_intervention, allow_external_agents) |
| `--owner` | string | 自動解決 | 送信先 owner の actor ID |
| `--dry-run` | bool | `false` | プランニングプレビューのみ実行 |
| `--approve` | TOKEN | - | プレビューで取得した approval token を指定して submit |
| `--missing-only` | bool | `false` | --dry-run と併用。不足情報のみ表示 |
| `--json` | bool | `false` | JSON 形式で出力 |

<Callout type="info">
`human_intervention=required` の制約を持つビジョンは、先に `--dry-run` でプレビューを確認し、表示される `--approve` トークンを使って submit する必要があります。
</Callout>

<Terminal title="例: ビジョンを dry-run で確認してから submit">
```bash
# Step 1: プレビュー
starweft vision submit --title "新機能開発" --text "..." --dry-run

# Step 2: approve トークンで submit
starweft vision submit --title "新機能開発" --text "..." --approve <TOKEN>
```
</Terminal>

| `--constraint` | key=value[] | - | 制約条件 (複数指定可) |

## stop

プロジェクトまたはタスクツリーの停止命令を発行します。**principal ロール専用**です。

```bash
starweft stop --reason-code <CODE> --reason <TEXT> --yes [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--project` | string | - | 停止対象のプロジェクト ID。--task-tree と排他 |
| `--task-tree` | string | - | 停止対象のタスクツリールート ID。--project と排他 |
| `--reason-code` | string | - | 必須。停止理由コード |
| `--reason` | string | - | 必須。停止理由のテキスト |
| `--yes` | bool | `false` | 必須。停止を確認 |
| `--json` | bool | `false` | JSON 形式で出力 |

<Callout type="warning">
`--yes` フラグなしでの実行は `[E_CONFIRMATION_REQUIRED]` エラーになります。停止は取り消しできない操作です。
</Callout>

<Terminal title="例: プロジェクトを停止">
```bash
starweft stop --project proj_01ABCDEF --reason-code user_requested \
  --reason "目標達成のため停止" --yes
```
</Terminal>

## wait

指定した条件が満たされるまでポーリングで待機します。

```bash
starweft wait [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--vision` | string | - | 対象 vision ID |
| `--project` | string | - | 対象プロジェクト ID |
| `--task` | string | - | 対象タスク ID |
| `--stop` | string | - | 対象 stop ID |
| `--until` | string | - | 待機条件 (下記参照) |
| `--timeout-sec` | u64 | `120` | タイムアウト (秒) |
| `--interval-ms` | u64 | `500` | ポーリング間隔 (ミリ秒) |
| `--json` | bool | `false` | JSON 形式で出力 |

`--until` に指定できる値はスコープによって異なります。

<Accordion title="--until の有効な値">

- **`--vision`**: `project_created`, `active`, `stopped`
- **`--project`**: `available`, `active`, `approval_applied`, `stopping`, `stopped`
- **`--stop`**: `ordered`, `stopping`, `stopped`

</Accordion>

<Callout type="info">
`--vision`, `--project`, `--task`, `--stop` はいずれか 1 つだけ指定してください。
</Callout>

## backup

ノードの全データのバックアップと復元を行います。

### backup create

データディレクトリを署名付きバンドルとしてバックアップします。

```bash
starweft backup create --output <PATH> [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--output` | path | - | 必須。バックアップ出力先ディレクトリ |
| `--force` | bool | `false` | 既存の出力先を上書き |

### backup restore

バックアップバンドルからデータを復元します。

```bash
starweft backup restore --input <PATH> [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | 復元先データディレクトリのパス |
| `--input` | path | - | 必須。バックアップバンドルのパス |
| `--force` | bool | `false` | identity 不一致でも強制復元 |

<Callout type="warning">
異なるノードのバックアップを復元する場合は `--force` が必要です。identity の不一致が検出されると `[E_BACKUP_IDENTITY_MISMATCH]` エラーになります。
</Callout>

## repair

データの修復操作を行います。

### repair rebuild-projections

タスクイベントログからプロジェクション (プロジェクト・タスクのスナップショット) を再構築します。

```bash
starweft repair rebuild-projections [--data-dir <PATH>]
```

### repair resume-outbox

pending 状態の outbox メッセージを再キューします。

```bash
starweft repair resume-outbox [--data-dir <PATH>]
```

### repair list-dead-letters

配送不能メッセージ (dead letters) を一覧表示します。

```bash
starweft repair list-dead-letters [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--limit` | usize | `20` | 表示件数の上限 |
| `--json` | bool | `false` | JSON 形式で出力 |

### repair reconcile-running-tasks

ランタイム停止後に running 状態のまま残ったタスクを停止状態に修正します。

```bash
starweft repair reconcile-running-tasks [--data-dir <PATH>]
```

## openclaw

OpenClaw バイナリの設定を管理します。

### openclaw attach

OpenClaw バイナリのパスと設定を `config.toml` に書き込みます。

```bash
starweft openclaw attach --bin <PATH> [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--data-dir` | path | `~/.starweft` | データディレクトリのパス |
| `--bin` | string | - | 必須。OpenClaw バイナリのパスまたはコマンド名 |
| `--working-dir` | path | - | OpenClaw の作業ディレクトリ |
| `--timeout-sec` | u64 | - | タスク実行タイムアウト (秒) |
| `--enable` | bool | `false` | openclaw.enabled を true に設定 |

<Terminal title="例: OpenClaw をアタッチして有効化">
```bash
starweft openclaw attach --bin /usr/local/bin/openclaw --enable
```
</Terminal>

## registry serve

ピアディスカバリ用の HTTP レジストリサーバーを起動します。

```bash
starweft registry serve [OPTIONS]
```

| フラグ | 型 | デフォルト | 説明 |
|-------|-----|-----------|------|
| `--bind` | string | `127.0.0.1:7777` | バインドアドレス |
| `--ttl-sec` | u64 | `300` | ピアエントリの TTL (秒) |
| `--rate-limit-window-sec` | u64 | `60` | レートリミットウィンドウ (秒) |
| `--announce-rate-limit` | u64 | `300` | announce エンドポイントのレートリミット |
| `--peers-rate-limit` | u64 | `900` | peers エンドポイントのレートリミット |
| `--shared-secret` | string | - | HMAC 認証用の共有シークレット (STARWEFT_REGISTRY_SHARED_SECRET 環境変数も可) |
| `--shared-secret-env` | string | - | 共有シークレットを格納する環境変数名 |
| `--max-body-bytes` | usize | `65536` | リクエストボディの最大サイズ (bytes) |
| `--read-timeout-ms` | u64 | `5000` | 読み取りタイムアウト (ms) |
| `--write-timeout-ms` | u64 | `5000` | 書き込みタイムアウト (ms) |
| `--allow-insecure-no-auth` | bool | `false` | 認証なしでの非ループバックバインドを許可 (非推奨) |

<Callout type="warning">
非ループバックアドレスにバインドする場合、`--shared-secret` または `--shared-secret-env` が必須です。テスト目的でのみ `--allow-insecure-no-auth` を使用してください。
</Callout>
